# 可视化数据残缺问题与改进

## 问题描述

**现象**：
```
用户提问: "帮我分析这个家系图"
    ↓
[之前] DynamicVizGeneratorService (降级状态)
返回: {
  visualizationApplicable: true,
  selectedTemplate: { templateId: "pedigree_chart_v1" },
  extractedData: {},  // ← 问题：空的！
  visualizationData: {
    type: "pedigree_chart",
    data: {},  // ← 问题：没有家系数据
    elements: []
  },
  textAnswer: {
    mainAnswer: "家系图用于分析遗传病..."
    keyPoints: [...]
  }
}

前端渲染:
  ✅ 能显示UI框架（表格、输入框）
  ❌ 但没有实际数据，画不出家系图
```

---

## 根本原因分析

### 原因1：LLM降级时的数据生成能力

#### DynamicVizGeneratorService 的问题

```typescript
// 降级时的prompt
const prompt = this.buildGenerationPrompt({
  question,
  concept: conceptText,
  knowledgeContent: ragResult.results.map(r => r.content).join('\n\n'),  // ← 空字符串！
  recommendedTemplates: templateMatches,  // ← 空数组！
  userLevel
});

// LLM在没有足够上下文时，难以生成准确数据
const response = await this.llmService.structuredChat<DynamicVizResponse>(
  [{ role: 'user', content: prompt }],
  DYNAMIC_VIZ_RESPONSE_SCHEMA,
  { temperature: 0.3 }
);
```

**问题**：
- ❌ RAG检索失败 → `knowledgeContent` 是空字符串
- ❌ 模板匹配失败 → `recommendedTemplates` 是空数组
- ❌ LLM缺少上下文 → 难以准确提取数据
- ❌ `extractedData` 可能是空对象 `{}`
- ❌ `visualizationData.data` 缺少必要字段

#### DYNAMIC_VIZ_RESPONSE_SCHEMA 的要求

```typescript
const DYNAMIC_VIZ_RESPONSE_SCHEMA = {
  type: 'object',
  properties: {
    visualizationApplicable: { type: 'boolean' },
    selectedTemplate: {
      type: 'object',
      properties: {
        templateId: { type: 'string' },
        reason: { type: 'string' }
      },
      required: ['templateId', 'reason']
    },
    extractedData: {
      type: 'object',
      description: '从知识点中提取的数据'  // ← LLM需要填充这个
    },
    visualizationData: {
      type: 'object',
      description: '生成的可视化数据（必须包含知识点字段：keyPoints, understandingPoints, commonMistakes, checkQuestions）'
      // ↑ 这里的data必须包含具体内容
    },
    textAnswer: {
      type: 'object',
      properties: {
        mainAnswer: { type: 'string' },
        keyPoints: { type: 'array' },
        examples: { type: 'array' },
        commonMistakes: { type: 'array' }
      }
    }
  }
};
```

**关键问题**：
- `extractedData` 依赖LLM提取，降级时可能为空
- `visualizationData` 需要包含具体数据（如家系图中的个体、代数）

## 改进方案：VisualizationToolMatcherService

### 核心改进点

#### 改进1：不依赖Embedding

**之前的依赖**：
```typescript
// DynamicVizGeneratorService
const conceptEmbedding = await this.llmService.embed(concept);  // ← 需要API
const questionEmbedding = await this.llmService.embed(question);  // ← 需要API
```

**现在的实现**：
```typescript
// VisualizationToolMatcherService
private async findBestTool(question: UserQuestion, intentAnalysis: IntentAnalysis) {
  // 1. 关键词匹配（无需embedding）
  const keywordMatches = this.findKeywordMatches(question.content, toolType);

  // 2. 类别权重（无需embedding）
  const template = this.templateMatcher.getTemplate(toolType);
  const categoryScore = this.getCategoryScore(template.category);

  // 3. RAG检索（可选，降级时不影响）
  let ragMatches: VisualizationMatch[] = [];
  try {
    ragMatches = await this.visualizationRAG.retrieveByQuestion(question.content, 0.6, 5);
  } catch (error) {
    ragMatches = [];  // ← 降级处理
  }

  // 4. 综合评分
  const totalScore = (ragScore * 0.4) + (keywordScore * 0.4) + (categoryScore * 0.2);
}
```

**优势**：
- ✅ **不依赖embedding**：关键词和类别匹配足够
- ✅ **降级保护完善**：RAG失败时仍然可用
- ✅ **置信度计算完整**：多种评分维度

#### 改进2：提供默认参数

**之前的问题**：
```typescript
// DynamicVizGeneratorService降级时
{
  extractedData: {},  // ← 空！
  visualizationData: {
    data: {},  // ← 空！
    elements: []
  }
}
```

**现在的实现**：
```typescript
// VisualizationToolMatcherService
const suggestedParameters = {
  parent1: 'Aa',  // ← 使用默认值
  parent2: 'Aa',
  title: '庞氏方格'
};

return {
  matched: true,
  tool: {
    templateId: 'punnett_square_v1',
    schema: { ... },
    defaultValues: {  // ← 有完整的默认值！
      parent1: 'Aa',
      parent2: 'Aa',
      title: '庞氏方格'
    }
  },
  suggestedParameters,  // ← 传递给前端
  toolInfo: { ... }
};
```

**优势**：
- ✅ **默认值完整**：来自a2ui-templates.data.ts
- ✅ **前端可以渲染**：使用默认值即可显示
- ✅ **不依赖LLM提取**：数据来源可靠

#### 改进3：A2UI触发判断

**之前的问题**：
```typescript
// DynamicVizGeneratorService降级时
{
  visualizationApplicable: false,  // ← 可能错误判断
  applicableReason: '没有找到合适的模板'
}
```

**现在的实现**：
```typescript
// VisualizationToolMatcherService
const shouldTriggerA2UI =
  // 明确请求可视化
  explicitRequest ||

  // 高置信度
  intentAnalysis.confidence >= 0.7 ||

  // 核心工具且中等置信度
  (toolMatch.template.category === '核心工具' && intentAnalysis.confidence >= 0.5) ||

  // 综合评分足够
  toolMatch.totalScore >= 0.4;

return {
  matched: true,
  shouldTriggerA2UI: true,  // ← 多维度判断
  confidence: toolMatch.totalScore,
  suggestedParameters: { ... }
};
```

**优势**：
- ✅ **多维判断**：不只依赖单一条件
- ✅ **容错性强**：某个维度失败时仍可能触发
- ✅ **用户体验好**：避免误触发和漏触发

## 对比总结

### 之前系统（DynamicVizGeneratorService降级）

| 项目 | 状态 | 问题 |
|------|------|------|
| **LLM Chat** | ✅ 可用 | 可以生成文字回答 |
| **概念提取** | ✅ 可用 | 基于LLM Chat |
| **RAG检索** | ❌ 失败 | 返回空结果 |
| **模板匹配** | ❌ 失败 | 返回空列表（依赖embedding） |
| **数据提取** | ❌ 不完整 | extractedData为空或残缺 |
| **可视化数据** | ❌ 不完整 | visualizationData.data缺少字段 |
| **A2UI渲染** | ⚠️ 部分可用 | 有框架但无数据 |

**用户体验**：
- ⚠️ 能看到文字回答
- ⚠️ 能看到UI框架（如家系表的表格）
- ❌ 但无法看到完整的可视化（如家系图中的个体关系）

### 现在系统（VisualizationToolMatcherService）

| 项目 | 状态 | 改进 |
|------|------|------|
| **LLM Chat** | ✅ 可用 | 不变 |
| **概念提取** | ✅ 可用 | 不变 |
| **RAG检索** | ⚠️ 降级 | 返回空结果，但不影响匹配 |
| **工具匹配** | ✅ 可用 | 关键词+类别匹配，不依赖embedding |
| **数据提取** | ✅ 完整 | suggestedParameters来自默认值 |
| **可视化数据** | ✅ 完整 | 使用a2ui-templates.data.ts的完整schema |
| **A2UI渲染** | ✅ 完整可用 | 有框架且有数据 |

**用户体验**：
- ✅ 能看到文字回答
- ✅ 能看到完整的可视化（有默认数据）
- ✅ A2UI可以正常渲染和交互

## 改进总结

### 核心改进

| 方面 | 之前 | 现在 |
|------|------|------|
| **数据来源** | 依赖LLM提取 | 使用a2ui-templates默认值 |
| **Embedding依赖** | 强依赖 | 不依赖（可选） |
| **降级影响** | 数据残缺 | 仍可用（关键词+类别） |
| **A2UI触发** | 可能错误判断 | 多维度准确判断 |
| **用户体验** | 部分可用 | 完全可用 |

### 技术优势

**VisualizationToolMatcherService**：
1. ✅ **关键词匹配**：30+关键词映射，快速准确
2. ✅ **类别权重**：核心>重要>原理>技术
3. ✅ **降级保护**：RAG失败时仍然可用
4. ✅ **默认值完整**：来自a2ui-templates.data.ts
5. ✅ **多维判断**：明确请求+置信度+类别+评分
6. ✅ **灵活配置**：可以调整权重和阈值

### 用户价值

**之前**：
- ⚠️ 有时能回答，但可视化不完整
- ⚠️ 需要用户手动补充数据
- ⚠️ 体验不一致（有时有UI，有时没有）

**现在**：
- ✅ 总是有数据可用的可视化（使用默认值）
- ✅ A2UI触发判断更准确
- ✅ 降级时仍能提供基础功能
- ✅ 用户体验一致

## 总结

### 问题根源

**之前的可视化数据残缺原因**：
1. ❌ **依赖Embedding API**：失败时无法匹配模板
2. ❌ **LLM降级时缺少上下文**：无法提取准确数据
3. ❌ **extractedData依赖LLM**：降级时为空
4. ❌ **A2UI触发判断不准确**：单一维度判断

### 现在的改进效果

**VisualizationToolMatcherService**解决了所有问题：
1. ✅ **不依赖Embedding**：关键词和类别匹配足够
2. ✅ **降级时仍可用**：RAG失败不影响工具匹配
3. ✅ **提供默认值**：从a2ui-templates.data.ts获取
4. ✅ **多维A2UI触发**：明确请求+置信度+类别+评分
5. ✅ **数据完整性**：suggestedParameters来自可靠来源

**用户体验提升**：
- ⚠️ 之前 → ✅ 现在：总是有数据可用
- ⚠️ 之前 → ✅ 现在：可视化更完整
- ⚠️ 之前 → ✅ 现在：A2UI触发更准确
