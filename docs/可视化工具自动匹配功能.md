# 可视化工具自动匹配功能文档

## 功能概述

在AI速通聊天系统中，实现了智能的可视化工具匹配功能，能够根据用户提出的遗传学相关问题，自动从现有的多个可视化遗传学分析工具模板中选择最适合的工具。

## 实现流程

### 1. 用户提问解析
系统接收用户在AI速通聊天中的提问内容。

### 2. 语义分析和意图识别
通过 `VisualizationToolMatcherService` 对用户提问进行语义分析和意图识别：
- 判断用户是否有可视化分析的需求
- 识别关键词和概念
- 计算可视化意图的置信度

### 3. 工具匹配
根据分析结果，从可视化遗传学分析工具库中匹配最相关的工具：
- **RAG匹配**：使用 `VisualizationRAGService` 进行语义相似度检索
- **关键词匹配**：基于预定义的关键词进行匹配
- **类别权重**：核心工具 > 重要方法 > 原理演示 > 实验技术

综合评分计算：
```typescript
totalScore = (ragScore * 0.4) + (keywordScore * 0.4) + (categoryScore * 0.2)
```

### 4. A2UI触发判断
判断是否需要触发A2UI流程：
- 用户明确请求可视化（如"帮我画图"、"生成图表"）
- 意图置信度 >= 0.7
- 核心工具且置信度 >= 0.5

## API端点

### 1. 匹配可视化工具
**端点**: `POST /agent/visualize/match-tool`

**请求体**:
```typescript
{
  question: string;        // 用户问题内容（必需）
  concept?: string;         // 当前概念（可选）
  userLevel?: 'beginner' | 'intermediate' | 'advanced';  // 用户水平（可选）
}
```

**响应**:
```typescript
{
  success: boolean;
  message: string;
  result: {
    matched: boolean;
    tool: A2UITemplate | null;
    toolId: string | null;
    toolName: string;
    confidence: number;              // 匹配置信度 0-1
    intentAnalysis: {
      needsVisualization: boolean;
      confidence: number;
      reasoning: string;
      keywords: string[];
      concepts: string[];
    };
    suggestedParameters: Record<string, any>;
    shouldTriggerA2UI: boolean;  // 是否应该触发A2UI
    toolInfo: {
      templateId: string;
      visualizationType: string;
      complexity: 'low' | 'medium' | 'high';
      category: string;
      schema: object;
      defaultValues: object;
    } | null;
  }
}
```

### 2. 获取所有可用工具
**端点**: `GET /agent/visualize/tools`

**响应**:
```typescript
{
  total: number;
  tools: Array<{
    templateId: string;
    visualizationType: string;
    complexity: 'low' | 'medium' | 'high';
    category: string;
    schema: {
      type: string;
      required: string[];
    }
  }>
}
```

### 3. 按类别获取工具
**端点**: `GET /agent/visualize/tools/category/:category`

**参数**: `category` - 类别名称（如"核心工具"、"重要方法"等）

**响应**:
```typescript
{
  category: string;
  count: number;
  tools: Array<{
    templateId: string;
    visualizationType: string;
    complexity: 'low' | 'medium' | 'high';
    schema: {
      type: string;
      required: string[];
    }
  }>
}
```

## 意图识别规则

### 可视化相关关键词
以下关键词会触发可视化需求检测：
- 可视化、图、图表、展示、演示、画、图解
- 分析、计算、推导、预测、验证、检查
- 杂交、测交、系谱、家系、概率、比例
- 卡方、连锁、基因定位、遗传图谱、三点测交
- 细菌接合、数量性状、染色体畸变、孟德尔
- 庞氏、punnett、遗传病、携带者

### 置信度等级
- **高置信度 (≥0.7)**: 检测到3个以上相关关键词，或用户明确请求可视化
- **中等置信度 (0.5-0.7)**: 检测到2个相关关键词
- **低置信度 (0.3-0.5)**: 检测到1个相关关键词
- **无可视化需求 (<0.3)**: 未检测到相关关键词

### A2UI触发条件
满足以下任一条件即触发A2UI：
1. 用户明确请求可视化（如"帮我画图"、"生成图表"）
2. 意图置信度 >= 0.7
3. 核心工具类别且置信度 >= 0.5
4. 匹配置信度 >= 0.4

## 前端集成示例

### 示例1：用户提问自动匹配工具
```typescript
async function handleUserQuestion(question: string) {
  // 调用工具匹配API
  const response = await fetch('/agent/visualize/match-tool', {
    method: 'POST',
    body: JSON.stringify({
      question,
      userLevel: 'intermediate'
    })
  });
  
  const result = await response.json();
  
  if (result.success && result.result.shouldTriggerA2UI) {
    // 显示匹配的工具信息
    console.log(`推荐工具: ${result.result.toolName}`);
    console.log(`置信度: ${result.result.confidence}`);
    console.log(`理由: ${result.result.intentAnalysis.reasoning}`);
    
    // 触发A2UI流程
    const a2uiConfig = {
      type: 'card',
      id: 'viz_' + Date.now(),
      children: [{
        type: `ahatutor-${result.result.tool.visualizationType}`,
        id: 'component',
        properties: {
          ...result.result.tool.defaultValues,
          ...result.result.suggestedParameters
        }
      }]
    };
    
    // 发送A2UI配置到渲染器
    dispatch(renderA2UI(a2uiConfig));
  }
}
```

## 使用场景

### 场景1：孟德尔定律问题
**用户提问**: "Aa x Aa 杂交的后代基因型比例是多少？帮我画个图"

**系统分析**:
- 关键词: ["杂交", "比例", "画", "图"]
- 概念: ["孟德尔", "基因型"]
- 置信度: 0.9（高）
- 推荐工具: `punnett_square_v1`
- 触发A2UI: ✅ 是

### 场景2：系谱图分析
**用户提问**: "这个家系图中，显性遗传还是隐性遗传？"

**系统分析**:
- 关键词: ["家系图", "显性", "隐性", "遗传"]
- 概念: ["遗传", "家系"]
- 置信度: 0.75（中高）
- 推荐工具: `pedigree_chart_v1`
- 触发A2UI: ✅ 是

### 场景3：卡方检验
**用户提问**: "我用Aa x Aa杂交，实际观察是100个显性、50个隐性，这符合孟德尔比例吗？"

**系统分析**:
- 关键词: ["杂交", "观察", "比例", "孟德尔"]
- 概念: ["卡方", "验证"]
- 置信度: 0.65（中）
- 推荐工具: `probability_distribution_v1`（含卡方检验）
- 触发A2UI: ✅ 是（核心工具）

### 场景4：纯文本提问
**用户提问**: "什么是DNA？"

**系统分析**:
- 关键词: ["DNA"]
- 概念: ["DNA"]
- 置信度: 0.2（低）
- 推荐工具: 无
- 触发A2UI: ❌ 否（置信度 < 0.3）

## 核心服务

### VisualizationToolMatcherService
负责整体工具匹配流程：
- 意图分析
- 工具匹配（RAG + 关键词 + 类别）
- A2UI触发判断

### VisualizationRAGService
提供语义检索能力：
- 基于向量数据库的相似度检索
- 支持问题到可视化类型的匹配

### TemplateMatcherService
提供模板管理：
- 管理所有A2UI模板
- 提供模板查询和匹配功能

## 配置说明

### 类别权重调整
可以在 `VisualizationToolMatcherService` 中调整类别权重：

```typescript
let categoryScore = 0;
if (template.category === '核心工具') {
  categoryScore = 10;  // 可调整此值
} else if (template.category === '重要方法') {
  categoryScore = 8;
}
```

### 匹配算法调整
可以在 `findBestTool` 方法中调整匹配算法：

```typescript
const totalScore = 
  (ragScore * 0.4) +        // RAG权重
  (keywordScore * 0.4) +     // 关键词权重
  (categoryScore * 0.2);      // 类别权重
```

```typescript
const keywordMap: Record<string, string[]> = {

```

## 注意事项

1. **可选服务注入**: `VisualizationToolMatcherService` 使用 `@Optional()` 装饰器，确保在不启用时系统仍可运行
2. **置信度阈值**: 根据实际使用情况调整置信度阈值，避免误触发或漏触发
3. **性能优化**: RAG检索可能耗时，考虑添加缓存机制
4. **错误处理**: 所有API端点都有完善的错误处理，返回友好的错误信息
5. **日志记录**: 关键步骤都有日志记录，便于调试和监控
